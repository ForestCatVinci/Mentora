Я строю EdTech платформу Mentoria Hub. Это централизованная платформа для учеников 8-11 классов в Казахстане и СНГ, которая заменяет Telegram-каналы образовательной организации Mentoria. Две ключевые функции: лента образовательных возможностей (посты менторов) и асинхронные курсы.

Стек: React 18 + TypeScript + Vite + Tailwind CSS на фронте, FastAPI на бэке, Supabase как БД и Storage, OpenAI API для авто-теггинга контента. Фронтенд деплоится на Vercel, бэкенд на Fly.io.

Начни с полного фундамента. Реализуй всё описанное ниже.

1. SUPABASE MIGRATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Создай SQL migration файл со следующими таблицами:

users
  id, created_at, email, full_name, avatar_url
  role: 'student' | 'staff' | 'admin'
  grade: int 8-11 (только для студентов)
  interests: text[]
  goals: text[]

mentors
  id, created_at
  user_id → users
  full_name, avatar_url, university, speciality, bio
  is_active: bool

posts
  id, created_at
  created_by → users
  title, description, image_url
  deadline_date: date (nullable)
  tags: text[]          ← генерируется OpenAI
  category: text        ← генерируется OpenAI
  grade_range: int[]    ← генерируется OpenAI
  location: text        ← генерируется OpenAI
  summary_ru: text      ← генерируется OpenAI
  is_published: bool
  saves_count: int

courses
  id, created_at
  created_by → users
  title, description, image_url
  category, difficulty: 'beginner' | 'intermediate' | 'advanced'
  grade_range: int[]
  is_published: bool

lessons
  id, course_id → courses
  title, description, video_url
  order_index: int

saved_posts
  id, user_id → users, post_id → posts, created_at

user_progress
  id, user_id → users, lesson_id → lessons, completed_at

RLS политики:
- студент видит только is_published=true посты и курсы
- staff может создавать и редактировать только свой контент
- admin имеет полный доступ ко всему

Эндпоинты:

POST /posts
  Принимает: title, description, image (multipart/form-data), deadline_date
  Логика:
    1. Сохраняет изображение в Supabase Storage, получает публичный URL
    2. Вызывает OpenAI gpt-4o-mini (temperature=0, response_format: json_object) с промптом:

    System:
    You are a content tagger for an educational platform for students
    grades 8-11 in Kazakhstan and CIS. Return ONLY valid JSON, no markdown,
    no explanation.

    User:
    Tag this educational post.
    Title: {title}
    Description: {description}

    Return exactly this JSON:
    {
      "tags": [],         // 3-6 specific keywords e.g. ["robotics", "MIT", "research"]
      "category": "",     // one of: competition, event, scholarship, summer_program,
                          // internship, research, hackathon, olympiad, course, other
      "grade_range": [],  // applicable grades from [8, 9, 10, 11]
      "location": "",     // one of: online, kazakhstan, international
      "summary_ru": ""    // 1-2 sentence summary in Russian for the student feed card
    }

    3. Записывает пост + AI-теги в Supabase одной операцией
    4. Возвращает созданный пост

GET /feed/{user_id}
  Возвращает посты отсортированные по матчинг-скору.

  Алгоритм скоринга для каждого поста:

    score = 0

    # релевантность тегов
    score += len(intersection(post.tags, user.interests)) * 3

    # соответствие классу
    if user.grade in post.grade_range:
        score += 4

    # соответствие целям
    goals_category_map = {
        "university_abroad": ["scholarship", "summer_program", "research"],
        "olympiads": ["olympiad", "competition"],
        "startup": ["hackathon", "event"],
        "research": ["research", "internship"]
    }
    for goal in user.goals:
        if post.category in goals_category_map.get(goal, []):
            score += 3
            break

    # срочность дедлайна
    if post.deadline_date:
        days_left = (post.deadline_date - today).days
        if days_left < 0:
            continue  # expired — не показываем
        elif days_left <= 14:
            score += 4
        elif days_left <= 42:
            score += 2

    # свежесть
    days_old = (today - post.created_at).days
    if days_old < 3:
        score += 3
    elif days_old < 7:
        score += 1

    # тайбрейкер
    sort by: score DESC, post.saves_count DESC

POST /posts/{id}/save    — сохранить пост, инкрементировать saves_count
DELETE /posts/{id}/save  — убрать из сохранённых, декрементировать saves_count

GET /courses             — список опубликованных курсов с уроками
POST /courses            — создать курс (только staff/admin)

Аутентификация:
  Supabase Auth, email/password.
  После логина читаем role из таблицы users.
  Protected routes: неавторизованные → редирект на лендинг.

Онбординг (обязателен для студентов):
  После первой регистрации показываем экран онбординга.
  Шаг 1: выбор класса (8, 9, 10, 11) — большие кликабельные карточки.
  Шаг 2: выбор интересов из предустановленного списка с возможностью мультиселекта:
    STEM, Business, CS, Math, Language, Research,
    Olympiads, Entrepreneurship, Science, Social Impact
  Шаг 3: выбор целей:
    Поступление за рубеж, Олимпиады, Стартап, Исследования
  Без завершённого онбординга лента недоступна — редиректим обратно на онбординг.

PostCard компонент:
  Показывает: image, title, summary_ru, первые 3 тега, category badge,
  deadline с цветовой индикацией:
    <= 7 дней  → красный бейдж "X дней осталось"
    <= 42 дней → жёлтый бейдж
    > 42 дней  → серый, просто дата
  Кнопка сохранить (иконка закладки, toggle).

Форма создания поста (только staff/admin):
  Поля: title, description (textarea), image upload, deadline через DatePicker.
  DatePicker: всплывающий календарь при клике на поле даты.
  После submit — вызов POST /posts, показываем лоадер пока OpenAI теггирует.
  После успеха — редирект на созданный пост.

Форма создания курса (только staff/admin):
  Поля: title, description, image, category, difficulty, grade_range (мультиселект).
  Уроки добавляются отдельно после создания курса.

4. ДИЗАЙН И UI
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Интерфейс должен быть современным, чистым и приятным — уровень реального продукта,
не шаблонный Bootstrap-вид. Целевая аудитория — школьники 8-11 классов, поэтому
UI должен быть живым, но не перегруженным.

Требования к дизайну:
  - Чистая типографика, хороший font pairing (Inter или аналог)
  - Карточки с subtle тенями и hover-эффектами
  - Плавные transitions (150-200ms)
  - Цветовая система: основной акцент фиолетовый (#7C3AED или близкий),
    нейтральные серые для фона, семантические цвета для бейджей
  - Адаптивный дизайн: мобильный + десктоп
  - Navbar с иконками и подписями, активный таб выделен акцентом
  - Empty states с иллюстрацией и призывом к действию
  - Loading скелетоны вместо спиннеров где возможно
  - Онбординг должен выглядеть как продуманный wizard, не как форма

Не делай:
  - Дефолтный Tailwind серый вид без кастомизации
  - Таблицы там где должны быть карточки
  - Мелкий шрифт и плотную вёрстку
